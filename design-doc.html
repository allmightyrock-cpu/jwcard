01 · 시스템 개요
🎯
목적 및 배경
왜 만들었는가

여호와의 증인 회중에서 사용하는 종이 구역카드(S-13 기반)를 디지털화한 웹 앱입니다. 기존에는 장로가 종이 카드를 손으로 관리하고 전도인에게 직접 건네주는 방식을 사용했으나, 분실·오기·회수 지연·현황 파악 어려움 등의 문제가 있었습니다.

해결한 문제
  • 구역 배정·회수 실시간 동기화
  • 방문 기록 디지털 저장
  • S-13 양식 자동 생성·출력
  • 전시대봉사 팀 편성·배정표 공유
  • 주소 오류 수정 요청 체계화
  • 현황 대시보드로 즉각 파악
핵심 설계 원칙
  • 월 비용 0원 (Firebase + Cloudflare 무료 한도)
  • 설치 불필요 — 브라우저 접속만으로 사용
  • 스마트폰 홈 화면 설치 (PWA)
  • 오프라인에서도 마지막 상태 유지
  • 개인정보 최소 수집 원칙
  • 프레임워크 없음 — 순수 HTML/JS

📄
S-13 구역 배정 기록부 자동 생성
  • 공식 JW S-13-KO 양식(A4 세로, 20행/페이지) — 실제 용지 크기에 맞춰 인쇄됨
  • 데이터 소스: territories[*].cycleHistory 배열
  • 기간 필터: 날짜 범위 지정 (기본 최근 6개월)
  • 봉사연도 자동 계산 — 9월 시작 기준 (예: 2025.9 ~ 2026.8 = 2025/26년도)
  • 진행 중인 배정도 포함 (completedDate 칸 공백)
  • 단축 날짜 형식: YY.MM.DD
🛒
전시대봉사 주간 자동 생성
  • 주간 기준: 월요일~일요일 (일요일은 같은 주 마지막 날로 처리)
  • 팀 편성 버튼 클릭 시 해당 주 일정이 없으면 Firestore에 자동 생성
  • 주간 키(wkStr): 로컬 날짜 기준 YYYYMMDD (UTC 오프셋 버그 방지)
  • 6개 팀(A~F) × 최대 6개 조 × 조당 3명 구성

10 · 기술 스택
분류기술선택 이유
UI 프레임워크 없음 (Vanilla JS) 의존성 없이 장기 유지 가능. 번들러·빌드 도구 불필요. 임베디드 HTML 단일 파일 배포 가능
데이터베이스 Firebase Firestore v10 실시간 동기화(onSnapshot), 무료 한도 충분, 서버 불필요, 오프라인 지원
인증 Firebase Auth 익명 별도 계정 없이 Firestore 보안 규칙 통과. 역할 제어는 클라이언트 로직으로 처리
호스팅 Cloudflare Workers / Pages 무료 CDN, HTTPS 자동, 글로벌 엣지, 정적 파일 업로드만으로 배포 완료
지도 Naver Maps API 한국 주소 정확도, 한글 도로명 검색 지원
엑셀 처리 SheetJS (XLSX) + JSZip 서버 없이 브라우저에서 .xlsx 파싱·생성
이미지 캡처 html2canvas v1.4.1 팀 배정표 DOM → PNG 변환 (카카오 공유용)
PWA Service Worker (sw.js) 오프라인 캐시, 홈 화면 설치, 업데이트 알림 배너
카카오 공유 Kakao JS SDK v2.7.2 카카오톡 피드 공유 (이미지 업로드 → 피드 전송)

12 · 보안 모델
Firestore 보안 규칙
// 익명 인증된 사용자만 읽기·쓰기 허용 rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
앱 로드 시 자동으로 signInAnonymously()를 호출하여 토큰을 획득합니다. 브라우저 외부에서의 무단 API 접근을 차단합니다.
역할 기반 접근 제어 (RBAC)
  • Firestore 규칙은 단순 인증만 처리
  • 역할별 UI 표시·숨김은 admin.js의 _applyRoleUI()가 담당
  • 세션은 localStorage에 저장 (4시간 유효)
  • 관리자 비밀번호·PIN은 평문 저장
    (내부 회중 전용 시스템, 외부 공개 불필요)
개인정보 보호
  • 방문 기록: 코드 선택 방식 — 세대주 정보 수집 없음
  • 전도인 동의 절차 내장 (consentGiven)
  • config.js는 git 제외 (.gitignore)
  • 구역 데이터는 회중 전용 서버에만 존재
⚠️ Firestore 규칙이 if true (모두 허용)로 설정된 경우 외부 접근이 가능합니다. 반드시 if request.auth != null로 설정하고, Firebase Authentication에서 익명 로그인을 활성화하세요.
