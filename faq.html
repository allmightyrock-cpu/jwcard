config.js

처음 설치는 어디서 시작하나요?
공식 설치 절차는 install-guide.html에 통합되어 있습니다. 설치 중 Firebase 설정값을 붙여넣어 config.js를 만들 때는 setup.html을 사용하세요.
config.js는 GitHub에 올려도 되나요?
아니요. config.js는 로컬 배포 폴더에만 둡니다. 공개 저장소에는 config.example.js만 올라가야 합니다.

Firebase 연결

로그인이 안 되거나 연결 오류가 납니다.
Firebase Authentication에서 익명 로그인이 켜져 있는지, Firestore Database가 생성되었는지, 배포본의 firestore.rules를 게시했는지 확인하세요. 가장 빠른 확인은 설치 점검 도구를 실행하는 것입니다.
관리자 비밀번호를 모릅니다.
이 시스템은 고정 기본 비밀번호를 두지 않습니다. 최초 접속 시 관리자가 직접 비밀번호를 설정합니다. 이미 설정된 뒤 분실했다면 Firebase의 관리자 설정 문서를 확인하거나 별도 초기화 절차가 필요합니다.

지도와 위치

지도가 안 뜨거나 지도 SDK 오류가 납니다.
네이버 클라우드 Maps 애플리케이션의 Web 서비스 URL에 현재 배포 주소가 등록되어 있는지 확인하세요. https://...를 포함해야 하며, 도메인이 바뀌면 새 주소도 등록해야 합니다.
지도는 뜨지만 주소 위치가 안 잡힙니다.
주소 좌표 변환에는 Geocoding API가 필요합니다. 네이버 클라우드에서 해당 애플리케이션에 Geocoding이 활성화되어 있는지 확인하세요.

주소와 엑셀 업로드

도로명과 번지가 이상하게 붙어 보입니다.
엑셀의 도로명, 번지, 건물명, 세부주소 열이 올바르게 나뉘어 있는지 확인하세요. 업로드 뒤에는 관리자 화면의 데이터 정비 기능으로 일부 자동 보정이 가능합니다.

업데이트

파일을 수정했는데 화면이 바뀌지 않습니다.
Cloudflare에 새 배포를 했는지 확인하세요. PWA로 설치한 경우 새 버전 배너를 누르거나 브라우저 새로고침이 필요할 수 있습니다.
업데이트하면 기존 데이터가 사라지나요?
일반적인 파일 업데이트로는 사라지지 않습니다. 기록 데이터는 Firebase Firestore에 저장됩니다. 다만 큰 작업 전에는 Firestore 백업을 권장합니다.

개인정보 보호

이 시스템은 개인정보를 어떻게 줄이나요?
거주자의 이름, 연락처, 신상 정보는 입력하지 않는 구조입니다. 방문 결과는 정해진 코드로 기록하고, 전도인은 이름 중심으로만 사용합니다. 실제 운영 전에는 회중 상황에 맞는 보안 규칙과 접근 권한을 반드시 점검하세요.
