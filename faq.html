❓ 자주 묻는 질문 · 문제 해결

증상별로 원인과 해결 방법을 정리했습니다. 막히면 먼저 설치 점검(진단)을 실행해 보세요.

설치 / 설정
처음 설치는 어떻게 하나요?
간편 설치 도우미를 쓰는 게 가장 쉽습니다.
  1. setup.html을 열고 Firebase 콘솔에서 복사한 값을 붙여넣으면 config.js가 자동 생성됩니다.
  2. 받은 config.js를 배포 폴더에 넣고 Cloudflare에 재배포합니다.
  3. /admin.html 접속 → 최초 접속 시 관리자 비밀번호를 직접 설정합니다.
⚠️ Firebase 콘솔의 코드 스니펫(import…, const firebaseConfig…)을 통째로 붙여넣지 마세요. '구성(Config)'의 값만 필요합니다.
관리자 비밀번호가 1914로 로그인이 안 돼요
이 시스템에는 고정 기본 비밀번호가 없습니다. 최초 접속 시 직접 설정합니다.
  1. 이름은 비우고 원하는 비밀번호 입력 → 로그인
  2. "비밀번호 설정 모드"로 바뀌면 비밀번호·확인란에 같은 값(4자리 이상) 입력 → "비밀번호 설정 및 로그인"
이미 설정했는데 "비밀번호가 올바르지 않습니다"가 뜨면 저장된 값과 다른 것이니, 다른 관리자에게 확인하거나 초기화가 필요합니다.
지도 / 위치
지도가 안 뜨거나 "지도 SDK 로드 실패"가 나와요
거의 대부분 네이버 클라우드(NCP) 콘솔 설정 문제입니다.
  1. 배포 도메인 등록 — NCP 콘솔 → Maps 애플리케이션 → Web 서비스 URL에 현재 배포 주소를 정확히 등록 (예: https://○○○.pages.dev, http/https·www 유무·커스텀 도메인 모두). 반영에 몇 분 걸립니다.
  2. 키 종류 확인 — 입력한 값이 Client ID(ncpKeyId)가 맞는지 (Client Secret 아님).
💡 설치 점검을 실행하면 인증 실패인지 자동으로 알려줍니다.
지도는 뜨는데 위치(핀)가 안 잡혀요
지도(Dynamic Map)와 주소→좌표 변환(Geocoding)은 별도 API입니다.
  • NCP 콘솔에서 해당 애플리케이션에 Geocoding API가 이용 신청/활성화되어 있는지 확인하세요. (지도만 신청하면 위치가 안 잡힙니다)
  • 관리자 설정에만 네이버 키를 넣은 경우, 최신 버전으로 재배포되어 있어야 위치 변환이 동작합니다.
주소가 "도로명+번지"로 붙어서 이상하게 보여요
번지는 크게(앞), 도로명은 작게 표시되는 게 정상입니다. 붙어 보이면 업로드한 엑셀에서 도로명과 번지가 한 칸에 들어간 것입니다.
  • 엑셀의 도로명·번지 컬럼을 나눠 다시 업로드하면 가장 깔끔합니다.
  • 최신 버전에서는 자동 보정이 있어, 재배포 후 다음 접속부터 대부분 자동으로 분리되어 표시됩니다.
연결 / 로그인
로그인이 안 되거나 "연결 오류"가 나와요
Firebase 설정·규칙 문제일 가능성이 높습니다.
  1. 익명 인증 — Firebase 콘솔 → Authentication → 로그인 방법 → 익명 사용 설정.
  2. 보안 규칙 — Firestore 규칙에서 인증 사용자 읽기/쓰기 허용 (request.auth != null).
  3. config.js 값 — apiKey·projectId가 본인 프로젝트 값인지 확인. (설치 도우미로 재생성 가능)

업데이트 / 기타
수정/업데이트가 화면에 반영되지 않아요
  1. 파일을 고쳤다면 Cloudflare에 다시 배포(재업로드)했는지 확인하세요. 배포해야 반영됩니다.
  2. 모바일 앱(PWA)은 상단에 "🔄 새 버전 업데이트" 배너가 뜨면 눌러서 갱신하세요.
  3. 그래도 안 보이면 브라우저 새로고침 또는 앱을 닫았다 다시 여세요.
이 시스템은 개인정보를 어떻게 보호하나요?
거주자(집주인)의 개인정보를 처음부터 수집·저장하지 않습니다. 방문 결과는 '방문/부재' 같은 상태로만 기록되고, 거주자의 이름·연락처·신상을 입력하는 칸 자체가 없습니다. 구역은 주소(건물·지번) 단위로만 관리되며, 자료는 회중 전용 DB에 인증된 접근만 허용됩니다.
