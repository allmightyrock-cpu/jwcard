전자구역카드 시스템 설치 안내서

처음 설치하는 회중을 위한 공식 안내서입니다. 전체 흐름, 설정 도구, 설치 점검, 문제 해결을 한곳에 정리했습니다.

전체 흐름 설치 도구 설치 단계 문제 해결 업데이트

1. 전체 흐름

설치는 크게 “파일 받기 → Firebase 준비 → config.js 생성 → Cloudflare 배포 → 진단 → 관리자 초기 설정” 순서로 진행합니다.

1 파일 받기
GitHub에서 최신 배포본을 내려받습니다.
2 Firebase 준비
데이터베이스와 익명 인증을 설정합니다.
3 config.js 생성
설치 도우미로 설정 파일을 만듭니다.
4 Cloudflare 배포
폴더를 업로드해 접속 주소를 만듭니다.
5 설치 점검
진단 도구로 연결 상태를 확인합니다.
6 운영 시작
관리자 비밀번호와 전도인/구역 자료를 설정합니다.

2. 설치 중 사용하는 도구

⚙️ config.js 생성 도우미Firebase 설정값을 붙여넣어 config.js를 자동 생성합니다. 🩺 설치 점검 도구배포 후 Firebase, 지도, 앱 설정 상태를 점검합니다. ❓ 문제 해결 FAQ설치 후 운영 중 자주 생기는 문제를 확인합니다.
중요: config.js에는 각 회중의 Firebase 설정값이 들어갑니다. GitHub나 공개 게시글에 올리지 마세요. 공개 배포본에는 config.example.js만 포함하는 것이 원칙입니다.

3. 설치 단계

1
최신 배포본 다운로드
GitHub 저장소에서 최신 파일을 내려받고 압축을 풉니다. 폴더 안에 admin.html, publisher.html, cart.html, config.example.js가 있는지 확인합니다.
GitHub 저장소 열기
2
Firebase 프로젝트 만들기
Firebase 콘솔에서 새 프로젝트를 만들고 Firestore Database를 생성합니다. Authentication 메뉴에서 “익명” 로그인도 활성화합니다.

Firestore 보안 규칙은 배포본에 포함된 firestore.rules 파일 내용을 복사해 게시하세요. 규칙 게시 전에는 Firebase 콘솔의 Rules Playground로 주요 화면 접속이 가능한지 확인하는 것이 좋습니다.
3
config.js 만들기
Firebase 프로젝트 설정 화면의 웹 앱 SDK 설정값을 복사한 뒤 setup.html에 붙여넣습니다. 도우미가 config.js를 자동으로 만들어 줍니다.
Firebase 콘솔의 코드 스니펫 전체를 직접 붙여넣어 파일을 만들지 마세요. window.APP_CONFIG = { ... } 형태의 설정 파일만 필요합니다.
4
Cloudflare Pages에 배포
생성한 config.js를 배포 폴더에 넣은 뒤 Cloudflare Pages에서 폴더 전체를 업로드합니다. 배포가 끝나면 https://... 형태의 접속 주소가 발급됩니다.
5
설치 점검 실행
발급된 주소 뒤에 /diagnostics.html을 붙여 열고 점검을 실행합니다. Firebase 연결, 지도 키, 카카오 키, 앱 설정 상태를 확인할 수 있습니다.
설치 점검 열기
6
관리자 초기 설정
/admin.html로 접속해 관리자 비밀번호를 설정합니다. 이후 전도인 명단, 권한, 구역 자료, 회차 기준, 지도 API 등을 관리자 화면에서 설정합니다.

4. 선택 설정

네이버 지도 API

구역 지도와 주소 위치 표시가 필요하면 네이버 클라우드 콘솔에서 Maps API를 설정합니다. 발급받은 Client ID는 config.js에 넣거나 관리자 화면의 설정 메뉴에서 저장할 수 있습니다.

카카오 JavaScript 키

카카오톡 공유 기능이 필요할 때만 설정합니다. 카카오 개발자 사이트에서 Web 플랫폼 도메인에 Cloudflare 배포 주소를 등록하고 JavaScript 키를 발급받아 kakaoJsKey에 넣습니다.

5. 설치 중 자주 막히는 부분

config.js를 어디에 두어야 하나요?
배포 폴더의 최상위, 즉 admin.html과 같은 위치에 둡니다. 공개 GitHub 저장소에는 올리지 말고 로컬/배포 폴더에만 보관하세요.
로그인이 안 되거나 Firebase 연결 오류가 납니다.
Authentication에서 익명 로그인이 켜져 있는지, Firestore Database가 만들어졌는지, firestore.rules를 게시했는지, config.js의 projectId가 맞는지 확인하세요. 그래도 막히면 설치 점검 결과를 복사해 담당자에게 전달하면 원인 파악이 쉽습니다.
지도는 뜨지만 위치가 안 잡힙니다.
네이버 지도 표시와 주소 좌표 변환은 별도 API입니다. Naver Cloud에서 Maps뿐 아니라 Geocoding 사용 여부와 Web 서비스 URL 등록을 확인하세요.
수정했는데 화면에 반영되지 않습니다.
Cloudflare에 새 배포를 했는지 확인하세요. 모바일 앱처럼 설치한 PWA는 새 버전 배너를 눌러 갱신하거나 브라우저 새로고침이 필요할 수 있습니다.

6. 새 버전으로 업데이트

  1. GitHub에서 새 배포본을 내려받습니다.
  2. 기존에 사용하던 config.js를 새 폴더에 복사합니다.
  3. Cloudflare Pages에서 새 배포를 만듭니다.
  4. 데이터는 Firebase에 있으므로 일반적인 파일 재배포로는 기존 기록이 사라지지 않습니다.
업데이트 전에는 가능하면 Firestore 백업을 먼저 해 두는 것이 좋습니다.
