설치 단계
1
최신 버전 다운로드
GitHub에서 소스 파일 받기
  1. 아래 링크 접속 → 오른쪽 상단 초록색 [<> Code] 버튼 클릭
  2. [Download ZIP] 선택 → 다운로드 후 압축 해제
  3. 폴더 안에 config.example.js 파일이 있는지 확인합니다.
🔗 GitHub 저장소 열기
💡 업데이트 알림 받기: GitHub 페이지 우측 상단 [Watch] → [Releases only] 선택하면 새 버전이 출시될 때 이메일로 알림이 옵니다.
2
Firebase 프로젝트 만들기
데이터베이스(Firestore) 생성 및 키 복사
  1. 🔥 Firebase 콘솔 열기 접속 → [프로젝트 만들기]
  2. 프로젝트 이름 입력 (예: jwcard-회중이름) → Google 애널리틱스 사용 안 함 → 만들기
  3. 좌측 메뉴 Firestore Database → [데이터베이스 만들기]
  4. 위치: asia-northeast3 (서울) 선택 → 프로덕션 모드로 시작
  5. 생성 완료 후 [규칙] 탭 → 아래 내용으로 교체 후 [게시]:
// Firestore 보안 규칙 (익명 인증된 사용자만 접근) rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
⚠️ allow read, write: if true는 사용하지 마세요. 앱이 익명 인증을 자동으로 처리하므로 위 규칙이 안전합니다.

Firebase 익명 인증 활성화
  1. 좌측 메뉴 Authentication → [시작하기]
  2. Sign-in method → 익명 → 사용 설정 → 저장

앱 등록 및 키 복사
  1. 좌측 ⚙ 프로젝트 설정 → 일반 탭 → 하단 내 앱 → </> (웹) 클릭
  2. 앱 닉네임 입력 → 앱 등록
  3. SDK 설정 → 반드시 '구성(Config)'을 선택 → 표시되는 값들을 복사해 둡니다
    ⚠️ 'npm'이나 코드 스니펫(import…, const firebaseConfig…)이 아니라 '구성(Config)' 쪽 값만 필요합니다.
✅ 이 단계에서 apiKey, projectId, appId 등의 값을 얻게 됩니다.
3
config.js 만들기
본인 API 키를 설정 파일에 입력
💡 쉬운 방법 (추천) — 간편 설치 도우미
배포 폴더의 setup.html을 브라우저로 열고, STEP 2에서 복사한 내용을 그대로 붙여넣으면 올바른 config.js가 자동 생성·다운로드됩니다. 코딩/문법 걱정 없이 이게 가장 쉽습니다.
받은 config.js를 배포 폴더에 넣고 재배포하면 끝. (아래 수동 방법은 직접 편집을 원할 때만)
수동으로 만들기
  1. 다운로드한 폴더에서 config.example.js 파일을 복사
  2. 복사한 파일 이름을 config.js로 변경
  3. 메모장(또는 텍스트 편집기)으로 열어 STEP 2에서 복사한 값들을 입력:
// config.js — 이 파일은 절대 GitHub에 올리지 마세요! window.APP_CONFIG = { firebase: { apiKey: "AIzaSy...", authDomain: "프로젝트ID.firebaseapp.com", projectId: "프로젝트ID", storageBucket: "프로젝트ID.firebasestorage.app", messagingSenderId: "숫자...", appId: "1:숫자:web:영문숫자..." }, naverClientId: "네이버 Client ID", // STEP 5 참고 (선택) kakaoJsKey: "카카오 JavaScript 키" // STEP 6 참고 (선택) };
🚫 가장 흔한 설치 실패 원인 — 꼭 읽어주세요!
Firebase 콘솔의 코드 스니펫을 통째로 붙여넣으면 안 됩니다. 아래 줄들은 넣지 마세요:
import { initializeApp } from "firebase/app";
const firebaseConfig = { … };
const app = initializeApp(firebaseConfig);
이 코드를 넣으면 config.js가 깨져 앱이 전혀 작동하지 않습니다. 위 예시처럼 window.APP_CONFIG = { … }; 하나만 두고, firebase: { } 안에 값 6개만 채우세요. (초기화 코드는 앱이 알아서 처리합니다.)
⚠️ config.js에는 본인 Firebase 키가 들어있습니다. 절대 GitHub이나 카카오톡 등에 공유하지 마세요. (.gitignore에 자동 포함되어 실수로 GitHub에 올라가지 않습니다.)
4
Cloudflare Pages 배포
웹 앱을 인터넷에 공개 (무료)
  1. ☁️ Cloudflare 대시보드 열기 접속 → 회원가입(또는 로그인)
  2. 좌측 메뉴 Workers & Pages → Pages → [파일 직접 업로드]
  3. 프로젝트 이름 입력 (예: jwcard)
  4. 다운로드한 폴더를 통째로 드래그&드롭 — config.js가 포함되어 있는지 꼭 확인!
  5. [배포] 클릭 → 완료 후 발급된 주소 확인 (예: jwcard.pages.dev)
✅ 배포 완료! 발급된 주소를 회중 전도인들과 공유하세요.
이후 업데이트 시에도 동일한 방법으로 새 파일을 업로드하면 됩니다.
5
첫 로그인 및 기본 설정
비밀번호 변경 · 전도인 등록 · 구역 업로드
  1. 배포된 주소 뒤에 /admin.html을 붙여 관리자 화면 접속
  2. 최초 접속 시 이름은 비우고 원하는 관리자 비밀번호를 입력하면 "비밀번호 설정 모드"로 전환됩니다 → 비밀번호·확인란에 같은 값(4자리 이상)을 입력하고 "비밀번호 설정 및 로그인" → 이 값이 관리자 비밀번호로 저장되며 바로 로그인됩니다
  3. 이후 로그인은 방금 설정한 비밀번호로 진행합니다 (필요 시 설정 탭 → 비밀번호 변경에서 변경)
  4. 전도인관리 탭 → 전도인 이름 등록 및 역할 지정
  5. 엑셀 구역카드 파일이 있다면 구역관리 → 엑셀 업로드로 구역 데이터 가져오기

6
네이버 지도 API 설정
관리자 화면에서 구역 지도 표시
선택
설정하지 않아도 앱은 정상 동작합니다. 지도 기능만 비활성화됩니다.
  1. 🗺️ 네이버 클라우드 열기 접속 → 로그인
  2. AI·NAVER API → Application → Application 등록
  3. 서비스 선택: Maps → Web Dynamic Map + Geocoding 두 개 체크
  4. Web 서비스 URL에 배포 주소 입력 (예: https://jwcard.pages.dev)
  5. 등록 후 Client ID 복사 → config.js의 naverClientId에 붙여넣기
💡 admin.html → 설정 탭 → 네이버 지도 API에서도 변경 가능합니다. (서버 재배포 불필요)
7
카카오 JavaScript 키 발급 및 설정 (선택 — 안 해도 됩니다)
"주소 수정 요청"을 카카오톡으로 보내는 기능에만 사용됩니다
선택
💬 카카오 키를 설정하면 아래 기능들이 활성화됩니다:

  🏠 주소 수정 요청 — 전도인이 구역카드를 보다가 주소가 틀렸을 때 담당 봉사의 종에게 카카오톡으로 수정 요청을 바로 보낼 수 있습니다.

  🛒 전시대봉사 팀 배정표 공유 — 인도자가 팀 편성을 완료한 뒤 배정표 스크린샷을 팀원들에게 카카오톡으로 한 번에 전송할 수 있습니다.
⚡ 설정하지 않아도 앱은 100% 정상 동작합니다.
카카오 키가 없으면 위 기능이 "클립보드 복사" 방식으로 자동 대체됩니다. 급하면 설치 후 나중에 추가해도 됩니다.
① 카카오 개발자 계정 만들기 (이미 있으면 건너뛰세요)
  1. 💬 카카오 개발자 사이트 열기 접속 후 오른쪽 상단 [로그인] 클릭
  2. 평소에 쓰는 카카오톡 계정으로 로그인합니다.
    ※ 별도의 가입 없이 카카오톡 계정을 그대로 사용합니다.
  3. 처음 접속하면 개발자 등록 동의 화면이 나올 수 있습니다. 내용 확인 후 동의하고 진행합니다.
② 애플리케이션 만들기
  1. 로그인 후 상단 메뉴 [내 애플리케이션] → [애플리케이션 추가하기] 버튼 클릭
  2. 아래와 같이 입력합니다:
    • 앱 이름: 원하는 이름을 자유롭게 입력하세요 (예: 전자구역카드 또는 jwcard)
    • 사업자명: 회중 이름이나 본인 이름을 입력해도 됩니다
    • 카테고리: 아무거나 선택해도 됩니다
  3. [저장] 버튼 클릭 — 애플리케이션이 만들어지면 자동으로 앱 대시보드 화면으로 이동합니다.
③ 웹 플랫폼 등록하기 ← 반드시 해야 하는 중요한 단계!
⚠️ 이 단계를 건너뛰면 카카오 공유 기능이 작동하지 않습니다.
카카오는 등록된 도메인에서만 공유 기능을 허용하기 때문입니다.
  1. 앱 대시보드 좌측 메뉴에서 [앱 설정] → [플랫폼] 클릭
  2. [Web 플랫폼 등록] 버튼 클릭
  3. 사이트 도메인 입력창에 Cloudflare에서 발급받은 주소를 입력합니다:
    https://jwcard.pages.dev
    ※ 본인 Cloudflare 도메인으로 교체하세요. https://를 꼭 포함해야 합니다.
  4. [저장] 클릭 — "Web 플랫폼이 등록되었습니다" 메시지 확인
④ JavaScript 키 복사하기
  1. 좌측 메뉴 [앱 설정] → [앱 키] 클릭
  2. 4가지 키가 보입니다. 그 중 JavaScript 키 오른쪽에 있는 값을 복사합니다.
    앱 키 종류 안내:
    • REST API 키 — ❌ 이건 아닙니다
    • JavaScript 키 — ✅ 이게 필요한 키입니다! (32자리 영문+숫자)
    • Admin 키 — ❌ 이건 아닙니다
    • Native 앱 키 — ❌ 이건 아닙니다
⑤ config.js 에 입력하기
  1. 메모장으로 config.js를 열어 아래처럼 입력합니다:
// config.js window.APP_CONFIG = { firebase: { /* ... 기존 Firebase 값 유지 ... */ }, naverClientId: "기존 값 유지", kakaoJsKey: "방금 복사한_32자리_JavaScript_키" // ← 여기에 붙여넣기 };
  1. 파일 저장 → Cloudflare에 재배포 (STEP 4와 동일한 방법으로 폴더 전체 다시 업로드)
  2. 앱에서 카카오톡 공유 버튼이 정상 동작하는지 확인합니다.
✅ 설정 완료! 이제 전도인이 구역카드에서 주소 오류를 발견하면 카카오톡으로 바로 알릴 수 있습니다. (전시대봉사 팀 배정표 공유는 이 키와 무관하게 항상 작동합니다.)
🚨 자주 하는 실수 모음
  • 사이트 도메인을 등록하지 않은 경우 → 공유 버튼을 누르면 아무 반응이 없습니다. ③번 단계를 다시 확인하세요.
  • JavaScript 키 대신 REST API 키를 입력한 경우 → 콘솔에 오류가 납니다. 키 종류를 다시 확인하세요.
  • config.js 저장 후 Cloudflare 재배포를 안 한 경우 → 변경 사항이 반영되지 않습니다.
  • https://를 빠뜨리고 도메인만 입력한 경우 → 플랫폼 등록이 제대로 되지 않습니다.

설치 완료 체크리스트
  • Firebase Firestore 데이터베이스가 생성되었다
  • Firestore 보안 규칙이 request.auth != null로 설정되었다
  • Firebase Authentication에서 익명 로그인이 활성화되었다
  • config.js에 Firebase 키가 모두 입력되었다
  • Cloudflare Pages 배포가 완료되어 주소가 발급되었다
  • 최초 접속 시 관리자 비밀번호를 직접 설정했다
  • 관리자 계정 이름이 등록되었다
  • (선택) 네이버 지도 API Client ID가 설정되었다
  • (선택) 카카오 JavaScript 키가 설정되었다 — 주소 수정 요청·팀 배정표 카카오톡 공유 활성화
  • 스마트폰에서 홈 화면에 앱이 추가되었다 (PWA)

    붙여넣기로 config.js를 자동 생성.
자주 묻는 질문
Q. 비용이 정말 없나요?
A. Firebase Firestore와 Cloudflare Pages는 소규모 회중 사용량에서 무료 한도 내에 들어옵니다. 데이터가 매우 많아지거나 접속자가 급증할 경우 소액 과금이 발생할 수 있으나 일반적인 회중 규모에서는 0원입니다.
Q. 구역카드 데이터가 삭제되지 않나요?
A. 모든 데이터는 Firebase Firestore에 저장됩니다. 앱 업데이트나 재배포와 무관하게 데이터는 유지됩니다.
Q. 기존 엑셀 구역카드를 그대로 사용할 수 있나요?
A. admin.html → 구역관리 → 엑셀 업로드를 통해 기존 엑셀 파일을 업로드할 수 있습니다. 샘플 파일은 다운로드 폴더 내 sample_excel/에 있습니다.
Q. 여러 명의 전도인이 동시에 접속해도 되나요?
A. Firebase 실시간 동기화 덕분에 여러 명이 동시 접속해도 항상 최신 상태가 유지됩니다.
Q. iOS(아이폰)에서도 사용 가능한가요?
A. 네. iOS Safari에서 접속 후 [공유 → 홈 화면에 추가]를 선택하면 앱처럼 설치되어 사용할 수 있습니다.
Q. 카카오 키를 설정했는데 공유 버튼을 눌러도 아무 반응이 없어요.
A. 대부분 사이트 도메인 등록 누락이 원인입니다. 카카오 개발자 사이트 → 내 애플리케이션 → 해당 앱 → [앱 설정] → [플랫폼] → [Web] 에 Cloudflare 배포 주소(https://...)가 등록되어 있는지 확인하세요.
Q. 카카오 개발자 사이트에서 어떤 키를 복사해야 하나요?
A. 앱 설정 → 앱 키에 4가지 키가 표시되는데, 그 중 「JavaScript 키」(32자리 영문+숫자)만 필요합니다. REST API 키·Admin 키·Native 앱 키는 이 앱에서 사용하지 않습니다.
Q. 전시대봉사 팀 배정표 공유는 어떻게 사용하나요? 카카오 키가 필요한가요?
A. 카카오 키는 필요 없습니다. 전시대봉사 앱(cart.html)에서 팀 편성된 일정 화면을 열면 팀 배정표 상단에 [💬 공유] 버튼이 있습니다. 누르면 배정표가 이미지로 만들어지고 — 휴대폰에서는 기본 공유 시트가 떠서 카카오톡·문자 등으로 보낼 수 있고, PC에서는 이미지가 다운로드되어 카카오톡 대화창에 첨부하면 됩니다. PC에서 다운로드되는 것은 정상 동작입니다(고장 아님).
