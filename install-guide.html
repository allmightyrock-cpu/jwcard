STEP 1 Firebase 프로젝트 설정 (데이터베이스)
Firebase 프로젝트 생성
  • 1
    firebase.google.com 접속 → Google 계정으로 로그인 → 우측 상단 [콘솔로 이동] 클릭
  • 2
    [프로젝트 추가] 클릭 → 프로젝트 이름 입력 (예: jwcard-회중이름) → [계속]
  • 3
    Google 애널리틱스 화면에서 "이 프로젝트에서 Google 애널리틱스 사용 설정"을 해제 → [프로젝트 만들기]
Firestore 데이터베이스 생성
  • 4
    왼쪽 메뉴 [Firestore Database] 클릭 → [데이터베이스 만들기]
  • 5
    "프로덕션 모드로 시작" 선택 → [다음] → 위치: asia-northeast3 (서울) 선택 → [완료]
  • 6
    [규칙] 탭 클릭 → 기존 내용을 모두 지우고 아래 내용으로 교체 후 [게시] rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if true; } } }
⚠
위 보안 규칙은 누구나 읽기/쓰기 가능한 설정입니다. 이 시스템은 관리자 비밀번호로 접근을 제어하므로 실제 서비스에 적합합니다. 다만 URL이 유출될 경우 데이터 접근 가능성이 있으니 URL은 필요한 인원에게만 공유하세요.
웹 앱 등록 및 설정값 확인
  • 7
    왼쪽 상단 톱니바퀴(⚙) → [프로젝트 설정] 클릭
  • 8
    [일반] 탭 → 아래로 스크롤 → "내 앱" 섹션 → [앱 추가] → 웹(</>) 아이콘 클릭
  • 9
    앱 닉네임 입력 (예: jwcard-web) → "Firebase 호스팅 설정"은 체크 해제 → [앱 등록]
  • 10
    화면에 표시되는 firebaseConfig 코드 전체를 복사해 메모해 두세요. const firebaseConfig = { apiKey: "AIzaSy...", authDomain: "your-project.firebaseapp.com", projectId: "your-project-id", storageBucket: "your-project.firebasestorage.app", messagingSenderId: "123456789", appId: "1:123456789:web:abc123" };
STEP 2 Naver Cloud Platform — Maps API 설정
Naver Cloud Platform 가입 및 Application 등록
  • 1
    ncloud.com 접속 → 우측 상단 [회원가입] → 개인 계정으로 가입
    ※ 신용카드 등록을 요구할 수 있으나, 무료 한도(300만 요청/월) 내에서는 결제가 발생하지 않습니다.
  • 2
    로그인 후 상단 메뉴 [콘솔] 클릭 → 왼쪽 메뉴에서 [AI·NAVER API] → [Maps] 클릭
  • 3
    [Application 등록] 클릭 → 이름 입력 (예: jwcard-map)
  • 4
    서비스 선택에서 아래 두 가지를 모두 체크하세요:
    ✅ Maps — Web Dynamic Map
    ✅ Maps — Geocoding
  • 5
    Web 서비스 URL 입력란에 배포할 Cloudflare 도메인 추가:
    예: https://my-jwcard.pages.dev
    ※ 도메인이 아직 없으면 임시로 http://localhost 입력 후 STEP 4 완료 후 수정
  • 6
    등록 완료 후 Client ID (ncpKeyId) 값을 복사해 메모해 두세요.
    예: abcd1234ef56 (발급받은 실제 값)
STEP 3 소스 코드 설정값 변경

텍스트 편집기(메모장, VS Code 등)로 아래 파일들을 열어 찾기·바꾸기(Ctrl+H)로 수정합니다.

① Firebase 설정 변경 (2개 파일)
  • 1
    js/admin.js 파일 열기 → 상단 5~12번 줄의 firebaseConfig 블록 전체를 STEP 1에서 복사한 내용으로 교체
  • 2
    publisher.html 파일 열기 → Ctrl+F로 const firebaseConfig 검색 → 해당 블록 전체(약 7줄)를 STEP 1에서 복사한 내용으로 교체
⚠
admin.js와 publisher.html 두 곳 모두 교체해야 합니다. 하나라도 빠지면 관리자와 전도인이 서로 다른 데이터베이스를 바라보게 됩니다.
② Naver Maps API 키 변경 (2개 파일)
  • 3
    admin.html 파일 열기 → Ctrl+H → 찾기: ncpKeyId=YOUR_NAVER_CLIENT_ID → 바꾸기: ncpKeyId=본인_Client_ID
  • 4
    publisher.html 파일에서도 동일하게 ncpKeyId=YOUR_NAVER_CLIENT_ID를 본인 Client ID로 교체
③ 회중 이름 설정
  • 5
    코드 수정은 불필요합니다. 배포 파일에는 이미 회중 이름이 제거되어 있습니다.
    Cloudflare 배포 후 admin.html → 로그인 → 설정 탭 → 회중 정보 → 회중명 입력 → 저장으로 설정하세요.
변경 파일 요약
파일변경 항목
js/admin.jsfirebaseConfig (5~12번 줄), 회중명
publisher.htmlfirebaseConfig, Naver Maps ncpKeyId, 회중명
admin.htmlNaver Maps ncpKeyId, 회중명
manifest.jsonname, short_name (회중명)
STEP 4 Cloudflare Pages 배포
Cloudflare 계정 생성 및 배포
  • 1
    cloudflare.com 접속 → [Sign Up] → 이메일·비밀번호 입력 → 이메일 인증 완료
    ※ 무료 플랜 선택 (Free plan)
  • 2
    대시보드 왼쪽 메뉴 [Workers & Pages] 클릭 → 우측 상단 [Create]
  • 3
    [Pages] 탭 선택 → "업로드로 배포" (또는 "Upload files") 선택
  • 4
    프로젝트 이름 입력 (예: my-jwcard) → 소스 파일 폴더 전체를 드래그 앤 드롭 (폴더 안의 모든 파일 선택)
  • 5
    [사이트 배포] 클릭 → 완료 후 자동 생성된 URL 확인
    예: https://my-jwcard.pages.dev
  • 6
    이 URL을 Naver Cloud의 Web 서비스 URL에 등록 (STEP 2-⑤에서 임시로 넣었다면 이 시점에 수정)
🔄
이후 파일 수정 시 재배포 방법:
Cloudflare 대시보드 → Workers & Pages → 해당 프로젝트 → [배포 관리] → [새 배포 만들기] → 수정된 파일 업로드
🔗
배포 후 URL 구조:
• 전도인 구역카드: https://my-jwcard.pages.dev/publisher.html
• 관리자 대시보드: https://my-jwcard.pages.dev/admin
STEP 5 시스템 초기 설정
관리자 최초 로그인 및 기본 설정
  • 1
    https://my-jwcard.pages.dev/admin 접속 → 최초 접속 시 "비밀번호 설정 모드"로 진입 → 관리자 비밀번호 설정 (4자리 이상, 기억하기 쉬운 숫자나 단어)
  • 2
    상단 메뉴 [설정] → 인도자 PIN, 봉사감독자 PIN 설정 (관리자와 다른 번호 권장)
  • 3
    [전도인관리] → 이름 입력 후 [전도인 추가] → 회중 전도인 전원 등록
  • 4
    [구역관리] → 우측 상단 [엑셀 업로드] 버튼 클릭
    구역번호·구역이름 입력 후 엑셀 파일 선택 → 미리보기 확인 → [저장]

  • 5
    전도인에게 아래 URL을 카카오톡·문자 등으로 공유:

자주 묻는 질문 (FAQ)
Q
지도가 표시되지 않아요
→ Naver Cloud Application의 "Web 서비스 URL"에 현재 접속하는 도메인이 정확히 등록되어 있는지 확인하세요. https:// 포함, 슬래시(/) 없이 입력해야 합니다.
Q
전도인이 접속하면 데이터가 안 보여요
→ publisher.html의 firebaseConfig가 admin.js와 동일한지 확인하세요. 프로젝트 ID가 다르면 서로 다른 DB를 바라봅니다.
Q
파일을 수정했는데 변경사항이 반영이 안 돼요
→ Cloudflare Pages에서 [새 배포 만들기]로 수정된 파일을 다시 업로드해야 합니다. 브라우저 캐시도 Ctrl+Shift+R로 강력 새로고침하세요.
Q
관리자 비밀번호를 잊었어요
→ Firebase 콘솔 → Firestore Database → admin 컬렉션 → config 문서 → password 필드를 직접 수정할 수 있습니다.
Q
PWA 앱 설치가 안 돼요
→ Android Chrome 또는 iOS Safari에서만 홈 화면 추가가 가능합니다. 카카오톡·네이버 앱 내 브라우저에서는 설치가 되지 않으니 Safari 또는 Chrome으로 열어야 합니다.
